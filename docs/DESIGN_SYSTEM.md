## 组件族

- `AppShell`：标题栏、固定侧栏、工作区边界
- `SidebarNav`：五个固定导航项、工作区状态、设置入口
- `Toolbar`：搜索、筛选、局部主操作
- `DataTable` / `DataRow`：代码仓、知识包、任务列表
- `AnalysisComposer`：范围输入、仓库/分支/知识/模式/模型选择与启动
- `FindingMasterDetail`：发现列表、事实/推断、双证据、修改建议
- `Modal`：添加代码仓、导入知识、忽略确认、设置
- `Status`：点、进度条、阶段文本；不使用无意义加载动画
- `InlineNotice`：成功、错误和校验反馈，保持在当前上下文

## 图标

统一使用单色、1.5–1.75px 描边、圆角线帽的线性图标。导航语义固定为：

- 新建分析：FilePlus2
- 代码仓：FolderGit2
- 知识库：BookOpen
- 分析任务：ListChecks
- 发现：SearchCode
- 设置：Settings

图标默认 16–18px；导航为 18px；不使用彩色图标容器。

## 动效与反馈

- 高频导航、筛选、键盘启动：即时切换，不做入场动画
- 按钮按压：`scale(0.97)`，120ms
- 弹窗与偶发浮层：180–220ms 强 `ease-out`，从 `scale(0.98)` 与透明度进入
- Hover 仅改变颜色/背景，120–160ms
- 仅动画 `transform` 与 `opacity`
- 支持 `prefers-reduced-motion`
- 分析运行必须展示真实阶段文本和进度，不只显示旋转图标

## 本地状态与关键路径

1. 添加并同步代码仓，成功/失败反馈保留在对应行。
2. 选择文档、检查知识草稿并发布不可变版本。
3. 在浮动输入区选择仓库、知识版本和范围，按钮或 `Ctrl+Enter` 发起分析。
4. 在任务页观察阶段与进度；完成后产生可追溯发现。
5. 在发现页并列查看代码证据与知识证据，确认后填写原因并忽略。
6. 在设置弹窗修改本地工作区和企业模型配置。

状态来自 Main 进程加载的工作区快照与任务/仓库事件；正式模式不使用示例数据或 `localStorage`。Electron 渲染进程不直接访问 Node.js、文件系统或凭据。
